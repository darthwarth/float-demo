Float Demo

The example boxes have been given a border to more easily see the outline of the containing div. The containing div houses an image and text.

Box one has a blue border

Box two has a red border

Box three has a green border

Box four has a black border

This box has the image floated left. Floated elements don't force the containing element to resize to accommodate them. For example, if the only thing in this element was the floated element, then the containing element would reduce in height to nothing, hence why the border of the containing element is currently only containing the text.
In this box the image is floated to the right, but because the float is still active from box one, the image in box one has overflowed over it.

In order to fix this any preceding floats need to be cleared.
Here the image is floated to the left, and the preceding floats have been cleared so the above elements don't spill over into this one. However, the image still spills out of the bottom of the containing element. This can be fixed by setting the display property.
This box has the image floated right. It also has all the previous floats (left and right) cleared, so the floats set above don't overflow into it, and it has its display property set so that none of the floated elements in the containing element spill out beyond its borders.

display: flow-root; is favoured over the more previously commonly used overflow: auto; due to some cases where certain side-effects were caused by overflow.
